How to remove Git submodules

11 Oct 2019

Here’s a simple guide on how to remove Git submodules. Lets say I want to remove a Git submodule called drafts.

  1. Edit .gitmodules to remove lines that are similar to the example below. 
    [submodule "drafts"]
  path = drafts
  url = git://github.com/emcorrales/drafts.git
  2. Stage the .gitmodules file. 
    git add .gitmodules
  3. Remove the submodule from the git index. Ensure that there is no trailing slash. We’re deleting the reference to the submodule and not the directory. ```bash

    Do it like this.

    git rm –cached drafts

Not like this.

git rm –cached drafts/

4. If you look at your repo's status, you'll see that the **drafts/** directory is now untracked.
```bash
$ git status
On branch dev
Your branch is up to date with 'origin/dev'.

Changes to be committed:
  (use "git reset HEAD <file>..." to unstage)

        modified:   .gitmodules
        deleted:    drafts

Untracked files:
  (use "git add <file>..." to include in what will be committed)

        drafts/
  1. Remove the untracked files left by the submodule. 
    rm -rf drafts
  2. Commit changes. 
    git commit -m "Remove draft submodule.".
  3. There are still references to the drafts submodule. Edit .git/config and remove lines that are similar to the example below. 
    [submodule "drafts"]
  url = git://github.com/emcorrales/drafts.git
  4. Remove the copy of the submodule’s git repo. 
    rm -rf .git/modules/drafts

    There you have it, you have succesfully removed the submodule from your local git repo.
